# Проверка релиза

## Автоматическая проверка состояния

После установки или восстановления запустите PowerShell от имени администратора:

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\build\test.ps1
```

Скрипт проверяет наличие конфигуратора, резервной копии, `state.json`, журнала, известного байта в точке патча и SHA-256 резервной копии. Успешный результат завершается строкой `RESULT: PASS`.

## Ручная проверка

- Установка на чистую Windows 11 с официальным пакетом.
- Проверка резервной копии.
- Проверка повторной установки.
- Подсветка: цвет, яркость, скорость, режим.
- Переназначение одной клавиши.
- Макрос на тестовой клавише.
- Сохранение и повторная загрузка INI-профиля.
- Переподключение USB.
- Восстановление оригинала через `build/restore.ps1`.
- Проверка запуска оригинального конфигуратора после отката.
- Переключение RU/EN с перезапуском после появления готового языкового payload.
- Удаление патча после появления деинсталлятора.

## Подтверждённый тест

На Windows 11 25H2 и клавиатуре `VID_30FA&PID_2350` подтверждены:

- установка исправленного EXE;
- повторная установка без повторной модификации;
- изменение цветов подсветки;
- переназначение клавиш;
- сохранение настроек в INI-файле;
- восстановление оригинального EXE.
